init 构造函数在struct与class中的区别

1. swift中的struct与class的异同

1.1 struct与class相同点

  • 定义属性用于存储值
  • 定义方法用于提供功能
  • 定义下标操作用于通过下标语法访问它们的值
  • 定义构造器（init）用于设置初始值
  • 支持扩展（extension）以增加默认实现之外的功能
  • 遵循协议（protocol）以提供某种标准功能

1.2 不同之处

  • 继承允许一个类继承另一个类的特征
  • 类型转换允许在运行时检查和解释一个类实例的类型
  • 析构器允许一个类实例释放任何其所被分配的资源
  • 引用计数允许对一个类的多次引用

以上主要是针对 class 来说的，struct 是没有这些的。

1.3 swift新增struct的目的猜测

Swift 之所有增加了 struct，主要有以下几点考虑：

安全性：值拷贝传递，没有引用计数；
内存：没有引用也就不会因为循环引用导致内存泄漏；
速度：值类型是直接在栈上分配，而不是在堆上，所以比 class 要快很多；

“堆”和“栈”并不是数据结构上的Heap跟Stack，而是程序运行中的不同内存空间；
栈是程序启动的时候，系统事先分配的，使用过程中，系统不干预；
堆是用的时候才向系统申请的，用完了需要交还，这个申请和交还的过程开销相对就比较大了；
栈是编译时分配空间，而堆是动态分配（运行时分配空间），所以栈的速度快；
从两方面来考虑：

  1. 分配和释放：堆在分配和释放时都要调用函数（MALLOC,FREE)，比如分配时会到堆空间去寻找足够大小的空间（因为多次分配释放后会造成空洞），这些都会花费一定的时间，而栈却不需要这些；
  2. 访问时间：访问堆的一个具体单元，需要两次访问内存，第一次得取得指针，第二次才是真正得数据，而栈只需访问一次；

拷贝：值类型是深拷贝，class 默认浅拷贝；
线程安全：自动线程安全的（无论哪个线程去访问都非常简单）；

1.4 struct和class的使用场景选择

  1. 不考虑应用的状态，也不存在对外的数据交互，就用 struct；
  2. 需要在多个地方修改 / 更新数据（共享数据），就用 class；
  3. 涉及到继承就用 class，没有就用 struct；
  4. 官方建议用 struct + protocol 来建模继承与共享；

1.5 示例说明和使用

struct MwebStruct {
    var name: String
    var age: Int
}
//MwebStruct 没有析构函数，但是我们可以用来初始化使用
let mweb = MwebStruct(name: "MWeb", age: 32)
//通过 init 构造器来初始化
let mwebinit = MwebStruct.init(name: "MWeb", age: 12)

print("mweb is = \(mweb.name), mweb age is \(mweb.age)")

//逐一成员构造器
//当我们没有显示指定至少一个init构造器时，swift会为我们添加一个默认构造器，该构造器的入参即是struct中的所有成员

//但是如果参数很多的话，我们只需要传入部分参数，我们该如何实现呢
//1.可选参数默认构造器

struct OptionalParamasWebStrcut{
    var name: String
    var age : Int?// 初始化时可选传参，不传时为nil
}
let opmweb = OptionalParamasWebStrcut(name: "OPPP") //可选参数默认构造器
print("oppp===\(opmweb.age ?? 0)")

//2.带默认参数的默认构造器

struct DefaultParamasStruct {
    var name : String
    //构造器不传参数时候，默认值为18
    var age = 18
}

let defaultMweb = DefaultParamasStruct(name: "Default")
print("default = \(defaultMweb.age)")

//3.扩展构造器
extension MwebStruct {
    //重载一个构造器
    //该构造器调用默认构造器
    init(name: String) {
        self.init(name: name, age: 16)
    }
}

let extenMweb = MwebStruct(name: "YUU")
print("exten == \(extenMweb.age)")

//class相关
class MWebClass{
    var name: String
    var age: Int
    //我们至少得显示指定一个构造器，我们要按照类似struct的逐一成员构造器那样，定义一个构造器
    
    //??如果只构造部分成员的构造器，则会如下报错 如下：
//    init(name: String) {
//        self.name = name //报错 Return from initializer without initializing all stored properties
//    }
    //解决方式
    /**
     1.将 age 设置为可默认为 nil 类型（等同于 2.1.2）；
     2. 定义 age 时就给定默认值（等同于 2.1.3）；
     3. 在构造器中给定默认值（如下代码所示）；
       如下
     */
    init(name: String) {
        self.name = name
        self.age = 10
    }
    
}

var mwebClass = MWebClass(name: "Mwebclassd")
print("mwebclass== \(mwebClass.name), == \(mwebClass.age)")

/**
 class 要求必需显式指定一个构造器，而 struct 则是由 swift 自动添加。同样，class 也可以通过扩展（extension）来重载构造器（注：扩展类的构造器，是需要便利的，即添加关键字：convenience）：
 */

extension MWebClass {
    convenience init(name: String , age: Int) {
        self.init(name: name)
        self.age = age
    }
}

var extenMwebClass = MWebClass(name: "ExtenClass", age: 100)

print("exten = \(extenMwebClass.name) age = \(extenMwebClass.age)")

//可失败构造器

var varirable : String? = nil


struct CanfailMwebStruct {
    var name : String
    init?(name: String) {
        if name == ""{
            return nil
        }
        self.name = name
    }
}

let canfailM = CanfailMwebStruct(name: "TT")
print("canfail====\(String(describing: canfailM))")
